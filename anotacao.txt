. Services Communication e DNS

O Kubernetes utiliza serviços para abstrair o acesso aos Pods, que são efêmeros (mudam de IP constantemente).

    K8s Service: Define uma política de acesso aos Pods.

    External to Service Communication: Como o mundo externo ou outros serviços fora do cluster se comunicam com suas aplicações.

    Kubernetes DNS: O cluster possui um servidor DNS interno (Kube-DNS ou CoreDNS). Cada serviço criado recebe um nome DNS automático, permitindo que os Pods se comuniquem pelo nome do serviço em vez de IPs.

2. ClusterIP Service

É o tipo de serviço padrão do Kubernetes.

    Propósito: Expõe o serviço em um IP interno do cluster.

    Acesso: O serviço só é acessível de dentro do cluster. É ideal para comunicação entre componentes internos (ex: uma aplicação Java acessando um banco Redis).

    Características: O serviço atua como um "labirinto" estável que sabe exatamente como chegar aos Pods, independentemente de quantos Pods existam ou de seus IPs individuais.

3. Gerenciamento de Portas

A configuração de portas é um dos pontos que mais gera confusão. A imagem esclarece a distinção:

    port: É a porta exposta pelo recurso Service. É por onde outros Pods tentarão acessar o serviço. (Obrigatório).

    targetPort: É a porta real onde a aplicação dentro do container está ouvindo. (Opcional: se omitida, assume o mesmo valor da port).
